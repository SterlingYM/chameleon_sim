Simulation without boundary conditions at source surface

For meeting 3/23/2020 (prelim) and 3/*/2020

Yukei Murakami, UCB: sterling@astro.berkeley.edu

Goals:

  • Finalize the simulation testing phase
  • Confirm the generalized case yields similar value to the approximated case
  • Simplify the code

Difference from Chameleon 3D Simulation (I):

  • No boundary condition for the inner sphere. This enables the simulation for weaker coupling where the value at the boundary is not at the equilibrium value. This allows us to simulate more complicated case (e.g. gaussian density distribution for the Cs source) as well.
  • Slower simulation. The solution tends to blow up when random value is given as an initial guess. To prevent this, we give the inner equilibrium value as an initial guess, which allows us to get the proper solution with a cost of simulation running time.

In this example, we define a class Rho which defines the density at each point in mesh. This method can be implemented either by defining two domains in mesh physical region file, or by manually setting the coordinate to re-map the domain in mesh (which is done by Inner() function in this case).

def solve(V,rho_solve,phi):
    v = fn.TestFunction(V)
    f = Lambda**5/phi**2
    g = fn.Constant(0)
    a = (fn.inner(fn.grad(phi)*scaling**2,fn.grad(v))+ rho_solve*v/M)/scaling*fn.dx
    L = f*v/scaling*fn.dx + g*v*fn.ds
    F = a-L
    fn.solve(F==0, phi, bcs)#, solver_parameters={'newton_solver' : {'linear_solver' : 'mumps'}})
    return phi

def importMesh(xml_file):
    physical_file = xml_file[:-4]+'_physical_region.xml'
    facet_file    = xml_file[:-4]+'_facet_region.xml'
    mesh       = fn.Mesh(xml_file)
    markers    = fn.MeshFunction("size_t", mesh, physical_file)
    boundaries = fn.MeshFunction('size_t', mesh, facet_file)
    dx = fn.Measure('dx', domain=mesh, subdomain_data=markers)
    V = fn.FunctionSpace(mesh, 'CG', 1)
    return mesh,markers,boundaries,dx,V

class Inner(fn.SubDomain):
    tol = 1e-14
    def inside(self,x,on_boundary):
        return x[0]**2 + x[1]**2 + x[2]**2 <= (R_sph + self.tol)**2

class Rho(fn.UserExpression):
    def __init__(self,materials,rho_0,rho_1,**kwargs):
        super().__init__(**kwargs)
        self.materials = materials
        self.rho_0 = rho_0
        self.rho_1 = rho_1
    def value_shape(self):
        return ()
    def eval_cell(self,values,x,cell):
        if self.materials[cell.index] == 0:
            values[0] = self.rho_0
        else:
            values[0] = self.rho_1

A few notes:

  • It is obvious that the shape near the source surface depends on the mesh size.
    TO ANALYZE THE FIELD NEAR THE SURFACE ACCURATELY, SIZE OF MESH MUST BE SMALLER THAN EXPECTED COMPTON LENGTH!!
    This of course comes with the cost of large computing time.
    TODO: The analysis of error as a function of the mesh size is required at some point.
  • As mentioned above, the solution does not converge properly if the initial guess is NOT a uniformly flat field at the expected equilibrium value. This is due to the property of the solver which is good at finding the nonzero curvature and is not good at choosing the uniform field as the $\nabla^2=0$ solution.
